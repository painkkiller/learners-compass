# Постоянные инструкции для агента (Learner’s Compass)

## 1. Базовый контекст проекта
- **Название**: Learner’s Compass — дашборд ученика.
- **Структура** (фиксирована для первой итерации):
- **Технологии**: React, Vite, Express, файл-заглушка вместо БД.

## 2. Маршрутизация по типам задач
- **UI / компоненты** → сначала смотреть `frontend/src/`, искать похожие компоненты.
- **API / данные** → смотреть `backend/` и `docs/api-design.md`.
- **Тесты** → писать/править в `tests/` параллельно с кодом.
- **Планирование** → перед изменениями читать `docs/iteration-1-plan.md`.

## 3. Ограничения
- **Не трогать** `scripts/` и `backend/` без явной необходимости (в первой итерации бэкенд — моковый, минимум изменений).
- **Не менять структуру проекта** (не создавать новые корневые папки) без подтверждения.
- **Не подключать реальную БД** — использовать моковые данные в `backend/mockData.js`.

## 4. Качество и тесты
- После каждого изменения UI → запускать `npm test -- --watch` (только релевантные тесты).
- После изменения бэкенда → запускать `npm test -- tests/backend/`.
- Если тестов нет для изменённой зоны — явно указать в отчёте.

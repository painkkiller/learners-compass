# План первой итерации: дашборд ученика

**Общая цель**: Создать работающую веб-страницу, где ученик видит:
- Список курсов, на которые он записан.
- Прогресс по каждому курсу (% завершения).
- Блок ближайших дедлайнов (заглушка).
- Кнопка «Следующий урок», ведущая на внешнюю ссылку-заглушку.

**Стратегия декомпозиции**: вертикальные срезы + послойно для моковых данных.

## Шаг 1. Моковые данные и API-заглушка
**Что делает агент**:  
Создаёт `backend/mockData.js` с массивом курсов (id, title, progress, nextLessonUrl, deadline) и простой Express-роут `GET /api/courses`.  
**Почему отдельно**: Без данных фронтенд не может отображать ничего. Это фундамент.  
**Ожидаемый результат**: `curl http://localhost:3001/api/courses` возвращает JSON.

## Шаг 2. Каркас React-приложения и роутинг (минимальный)
**Что делает агент**:  
Создаёт `App.jsx` с роутами: `/` (дашборд), `/profile` (заглушка). Добавляет простой Layout (шапка с названием).  
**Почему отдельно**: Отделяет структуру приложения от логики дашборда.  
**Ожидаемый результат**: Страницы переключаются, шапка видна.

## Шаг 3. Компонент списка курсов и отображение прогресса
**Что делает агент**:  
Создаёт `CourseList.jsx`, загружает данные с `/api/courses`, отображает карточки: название, прогресс (progress bar).  
**Почему отдельно**: Это центральный пользовательский сценарий — первый вертикальный срез.  
**Ожидаемый результат**: На главной странице видно 2-3 курса с полосками прогресса.

## Шаг 4. Блок ближайших дедлайнов
**Что делает агент**:  
Добавляет компонент `Deadlines.jsx`, получает из того же API массив дедлайнов (заглушка: сегодня, завтра, через 3 дня).  
**Почему отдельно**: Независимый блок, не влияет на список курсов. Легко проверить.  
**Ожидаемый результат**: Справа или сверху от списка курсов видны 3 дедлайна.

## Шаг 5. Кнопка «Следующий урок» для каждого курса
**Что делает агент**:  
В карточку курса добавляет кнопку, которая ведёт на `course.nextLessonUrl` (открывается в новом окне).  
**Почему отдельно**: Завершает основной сценарий «ученик идёт к следующему уроку».  
**Ожидаемый результат**: При клике открывается внешняя ссылка-заглушка.

## Шаг 6. Базовые тесты (критический путь)
**Что делает агент**:  
Пишет тесты: рендер `CourseList`, загрузка данных, клик по кнопке «Следующий урок» (проверка `window.open`).  
**Почему отдельно**: Тесты в конце, потому что интерфейс стабилизировался.  
**Ожидаемый результат**: `npm test` проходит хотя бы 3 базовых теста.

## Контрольные точки (как понять, что агент не ушёл в сторону)
- После шага 1: есть работающий эндпоинт (проверить в браузере или curl).
- После шага 3: страница показывает курсы (скриншот или live-проверка).
- После шага 5: кнопка ведёт на правильный URL (проверить в браузере).
- После шага 6: тесты зелёные.
